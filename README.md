# WordWriterID

An adaptive Markdown-like engine designed for technical and academic writing. 

## Key Features
- **Smart IDs:** Frictionless referencing using the `[..id]` syntax.
- **Hierarchical Sections:** Automatic `1.1.2` numbering for nested headers.
- **Academic Blocks:** Native, high-fidelity support for KaTeX math, CSV-to-Table parsing, and Prism.js syntax highlighting.
- **End-Matter Suite:** Automated bibliography, citations, and footnote management.

---

## 🛠️ Installation & Development

### 1. Prerequisites
Ensure you have [Node.js](https://nodejs.org/) and [Python](https://www.python.org/) installed.

### 2. Setup
Clone the repository and install dependencies:
```bash
git clone [https://github.com/shonczinner/wordwriterid.git](https://github.com/shonczinner/wordwriterid.git)
cd wordwriterid
npm install
```

### 3. Running the Live Demo
To test the engine in an interactive browser environment:
```bash
npm run demo
```
*Starts a local Python server at `port 8000` and opens the IDE.*

### 4. Static Build
To generate a standalone document from `test_input.txt` using Node.js:
```bash
npm run main
```

## 🚀 Live Previews

* **[Interactive Demo](https://shonczinner.github.io/wordwriterid/demo/demo.html)** **The Live Editor:** A real-time Markdown-like IDE. Use this to test the engine, experiment with syntax, and see dynamic rendering as you type.

* **[Static Build Example](https://shonczinner.github.io/wordwriterid/demo/test_output.html)** **The Final Product:** A self-contained, portable academic paper generated by `main.js`. This demonstrates how the engine bundles styles and content into a single, distribution-ready document.
